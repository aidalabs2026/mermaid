# 01. Vision & Scope

> PRD §1·§2·§3·§10 의 운영 검토판. 새 결정이 생기면 PRD §12 Decision Log 에 먼저 추가하고 본 문서를 동기화.

## 1. 한줄 비전

영문 글로벌 권역의 개발자·기술문서 작성자·DevOps 가 **mermaid syntax → 즉시 SVG 미리보기 + 학습 가이드** 를 한 곳에서 처리하는 프라이버시 우선 도구.

## 2. 왜 지금 이 프로젝트인가

`docs/04-service-candidates.md` 에는 mermaid_preview 가 정식 후보로 올라 있지 **않다**. 그러나 다음 두 필터를 통과해 후속 후보로 채택되었다.

### 2-1. 포트폴리오 필터 (`docs/00-portfolio-overview.md` §3) 통과 여부

| 필터 | 결과 |
|---|---|
| AdSense 정책 위반 컨텐츠 없음 | ✅ 개발자 도구, 위반 카테고리 해당 없음 |
| 한 사람이 2~3주에 MVP 가능 | ✅ json-beautifier 자산 재사용 4/5, mermaid.js 단일 라이브러리 |
| 검색 수요 검증 | ✅ "mermaid live editor", "mermaid online" 단일 키워드 월 수만~수십만 검색 |
| 클라이언트 처리 가능 (서버 비용 0) | ✅ mermaid.js 가 브라우저에서 SVG 렌더 |
| 카니발리즘 검토 | ✅ json-beautifier(JSON/XML/HTML) 와 도메인 무관, bidMaster(공공입찰) 와 무관 |
| 광고/AdSense 친화 톤 | ✅ 기술/교육 콘텐츠 |
| 영문 콘텐츠 운영 가능 | ✅ json-beautifier 와 동일 톤 |
| YMYL 회피 | ✅ 금융·의료 아님 |

### 2-2. 왜 후보 리스트(`04-service-candidates.md`) 에 없었는데 선정했는가

후보 리스트는 **검증된 키워드량** 위주로 구성되어 있고 mermaid 는 더 좁은 니치. 그러나:

- mermaid 는 **GitHub 가 README 에서 자동 렌더링** 한 이후 (2022~) 키워드 검색이 우상향.
- 1위권(`mermaid.live`) 이 SPA 단일 URL 이라 SEO 깊이를 잡기 어려움 → 우리가 다이어그램 타입별 랜딩으로 깊이를 만들면 롱테일 점유 가능.
- json-beautifier 의 Astro SSG + 클라이언트 라이브러리 + 가이드 + 용어 + 법적 페이지 구조를 **거의 1:1 재활용** 할 수 있음. 자산 재사용 4~5/5.
- 운영자(코드를 읽고 mermaid 다이어그램을 작성하는 개발자) 의 **흥미·도메인 친화성** 이 높아 가이드 작성 효율 높음 → AdSense 심사용 30+ 콘텐츠 확보 속도 빠름.

다음 후보 리스트 갱신 시 `04-service-candidates.md` §"개발자 도구" 카테고리에 정식 항목으로 추가할 것.

## 3. 타겟 (요약)

PRD §3 표를 그대로 따른다. 1차: 백엔드/풀스택 개발자 (README·블로그·ADR 작성 시 다이어그램 추가). 비타깃: 풀 SaaS UML 사용자, 비코드 드래그&드롭 도구 사용자.

## 4. MVP 범위 (확정)

PRD §5-1 의 F01~F13 을 MVP 로 채택. Phase 1 종료 조건:

- F01 라이브 프리뷰가 11종 mermaid 다이어그램 모두에 대해 동작
- F08 Share URL 이 Slack/Discord 미리보기에서 깨지지 않음
- F10 예시 갤러리에 11종 샘플 (각 30~80줄)
- 도구 페이지 + 다이어그램 타입 8 + 가이드 10+ + 용어 10+ + 법적 4 + 홈 = 30+ 페이지 색인

## 5. Out of Scope (PRD §10 발췌)

- 사용자 계정·로그인·클라우드 갤러리
- 협업 편집
- 유료 플랜
- 데스크톱 네이티브 앱
- 비-mermaid 다이어그램 엔진 풀 지원 (PlantUML 은 Phase 2 실험만)
- AI 기반 텍스트→다이어그램 자동 생성 (별도 프로젝트로 분리 검토)

## 6. 차별화 한 줄

> "mermaid.live 가 SPA 한 페이지로 끝낼 때, 우리는 11개 다이어그램 타입을 11개 SEO 깊은 랜딩으로 풀어 학습 + 도구 + 광고를 한 흐름으로 묶는다."
